# Context Loader - 自动上下文加载规则

> "泥板目录——用一块泥板，记录其他泥板放在哪。"

---

## 加载原则

**目标:** 在对话开始前，自动加载最相关的上下文，让 Genius 从一开始就"知情"

**原则:**
1. 分层加载：从基础身份到具体知识
2. 按需加载：根据用户输入关键词动态选择
3. 时效优先：优先加载最新的学习记录
4. 不重复加载：同一对话中已加载的不重复

---

## 基础加载 (每次对话必载)

**加载顺序:**

### Layer 0: 元知识系统
```
META-KNOWLEDGE.md
├── 系统架构理解
├── 当前操作层级 (A/B/C)
└── 加载规则本身
```

### Layer 1: 身份层
```
IDENTITY.md → SOUL.md → USER.md
```

**加载原因:**
- IDENTITY: 我是谁，我的判断风格
- SOUL: 我的核心价值观和判断力原则
- USER: 我在为谁服务，她的偏好

### Layer 2: 学习层 (最近10条)
```
learnings.md [最新的10条 active 规则]
```

**加载原因:**
- 确保最新的错误不再犯
- 新沉淀的规则立即生效

## 动态加载 (根据关键词匹配)

### 关键词 → 加载文件 映射表

| 关键词 | 加载文件 | 原因 |
|--------|----------|------|
| "日报" "daily" "report" | A-layer.md (Skill-001) + USER.md (日报标准) | 直接关联技能 |
| "竞品" "competitor" "对手" | A-layer.md (Skill-003) | 竞品分析技能 |
| "趋势" "trend" "判断" | A-layer.md (Skill-004) + SOUL.md (唯物辩证法) | 判断力原则 |
| "错了" "不对" "应该是" | B-layer.md + learnings.md | 进入学习模式 |
| "改进" "优化" "设计" | C-layer.md + B-layer.md | 改进机制 |
| "信息源" "source" "newsletter" | USER.md (信息源矩阵) | 信息检索配置 |
| "时效" "过期" "archive" | knowledge-ttl.yaml | TTL 规则 |
| "飞书" "feishu" "渠道" | 飞书配置 | 渠道相关 |

### 智能匹配规则

**规则 1: 精确匹配**
用户输入包含关键词 → 加载对应文件

**规则 2: 语义扩展**
```
"今天有什么新闻" → 匹配 "日报" → 加载 Skill-001
"帮我看看 OpenAI" → 匹配 "竞品" → 加载 Skill-003
"你这样不对" → 匹配 "错了" → 加载 B-layer.md
```

**规则 3: 组合加载**
用户输入包含多个关键词 → 加载所有匹配文件
```
"今天 OpenAI 有什么新闻，帮我判断一下趋势"
→ 加载: Skill-001 (日报) + Skill-003 (竞品) + Skill-004 (趋势)
```


## 加载优化

### 避免过度加载

**问题:** 加载太多文件会导致上下文过长，反而降低效率

**解决方案:**
1. **分层截断:**
   - 基础层: 全部加载
   - 动态层: 最多加载 3 个最匹配的文件
   - 深度层: 用户明确要求才加载

2. **摘要而非全文:**
   - 大文件加载摘要版本
   - 详细内容按需读取

3. **时效过滤:**
   - 只加载 active 状态的学习
   - 跳过已归档的内容

### 加载缓存

**同一对话内:**
- 已加载的文件不重复加载
- 仅追踪新增匹配

**跨对话:**
- 不缓存 (确保每次都有最新状态)

## 实现方式

### 方式 1: 文件引用 (推荐)

在对话开始时，系统提示中引用相关文件:
```
[系统提示]
当前加载的上下文:
- IDENTITY.md: [摘要]
- USER.md (日报标准): [摘要]
- learnings.md (最新3条): [摘要]
```

### 方式 2: 动态注入

根据关键词，动态将相关内容注入到 prompt:
```
用户提到了"竞品分析"，注入 A-layer.md Skill-003 的执行标准
```

### 方式 3: 检索增强 (RAG)

将所有文件向量化，根据用户查询语义检索最相关段落

## 与 A-B-C 层的关系

**A-Layer 调用时:**
- 加载对应技能的执行标准
- 加载相关的 learnings

**B-Layer 调用时:**
- 加载 learnings.md 全部内容
- 加载 B-layer.md 方法论

**C-Layer 调用时:**
- 加载全量上下文
- 加载历史模式报告
