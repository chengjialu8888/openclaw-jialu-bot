这是一个四层自生长的知识免疫系统，让 Genius 越用越聪明。

└─────────────────────────────────────────────────────────────┘

## 核心机制

### 1. 错误沉淀机制 (Error → Rule)

**流程：**
```
犯错 → 记录 → 分析根因 → 提炼规则 → 沉淀到 B-Layer → 下次自动调用
```

**文件位置：** `workspace/learnings.md`

**记录格式：**
```markdown
## [日期] 错误类型: [分类]

**场景:** [具体情境]
**错误:** [做了什么 wrong]
**根因:** [为什么错]
**修正规则:** [下次应该怎么做]
**关联技能:** [影响的 A-Layer 技能]
**生效状态:** [active / superseded by #X]
```

### 2. 知识时效性管理 (TTL System)

**规则：**
- 热点新闻: 7天后自动降级为"背景"
- 竞品动态: 30天后评估是否仍相关
- 战略判断: 90天后回顾验证
- 元规则: 永久有效，除非被新规则替代

**文件位置：** `workspace/knowledge-ttl.yaml`

### 3. A-B-C 自举循环

**A-Layer (做事层):**
- 具体任务执行
- 直接交付结果
- 技能清单: 日报生成、信息检索、分析判断

**B-Layer (改进层):**
- 监控 A-Layer 输出质量
- 沉淀 learnings
- 优化调用策略

**C-Layer (元层):**
- 评估 B-Layer 效率
- 优化学习机制本身
- 设计新的元规则

**自举循环：**
```
A 执行 → B 观察/沉淀 → C 评估/优化 → 改进 A → 循环
```

### 4. 自动上下文加载

**触发时机：** 每次对话开始时

**加载顺序：**
1. IDENTITY.md - 我是谁
2. SOUL.md - 我的核心判断原则
3. USER.md - 我在为谁服务
4. META-KNOWLEDGE.md - 系统规则
5. learnings.md - 最新的学习记录（最近10条）
6. 根据查询关键词动态加载相关内容

## 操作手册

### 当 AI 犯错时

用户可以说：
- "错了，应该是..."
- "记住这个错误..."
- "记下来：以后遇到这种情况..."

AI 自动：
1. 记录到 learnings.md
2. 分析根因
3. 提炼可执行规则
4. 确认规则生效

### 当知识过时

用户可以说：
- "这条信息过时了"
- "更新一下关于...的认知"
- "忘掉...，现在情况是..."

AI 自动：
1. 标记原知识状态
2. 记录新知识及生效时间
3. 更新检索索引

### 当系统需要优化

用户可以说：
- "改进你的工作方式"
- "优化日报生成流程"
- "设计一个更好的...机制"

AI 自动：
1. 进入 C-Layer 思考模式
2. 分析现有流程瓶颈
3. 提出改进方案
4. 用户确认后更新系统配置
