# B-Layer: Meta Layer (改进层)

> "伤其十指，不如断其一指" —— 识别问题，集中改进

---

## 本层定位

**做什么:** 监控 A-Layer，沉淀 learnings，优化调用策略
**不做:** 直接执行具体任务
**核心原则:** 每次错误都变成下次不再犯的能力

---

## 核心职能

### 1. 错误监控与沉淀

**触发条件:**
- A-Layer 上报错误
- 用户说"错了"
- 输出质量低于标准

**处理流程:**
```
接收错误 → 根因分析 → 提炼规则 → 记录到 learnings.md → 更新调用策略
```

**根因分析五问:**
1. 是什么错了？(事实)
2. 为什么错了？(原因)
3. 怎样才是对的？(修正)
4. 为什么之前没做对？(系统缺陷)
5. 如何确保下次不再犯？(规则)

---

### 2. 学习规则管理

**规则库位置:** `workspace/learnings.md`

**规则生命周期:**
```
draft → active → validated → permanent / superseded
```

**规则质量检查:**
- [ ] 是否可执行？
- [ ] 是否有明确触发条件？
- [ ] 是否关联到具体技能？
- [ ] 是否与现有规则冲突？

**规则冲突解决:**
- 后产生的规则 > 先前规则
- 被覆盖的规则标记为 `superseded by #[ID]`
- 冲突超过 3 次，上报 C-Layer

---

### 3. 调用策略优化

**策略模板:**

```markdown
## Strategy: [策略名]

**适用场景:** [触发条件]
**执行步骤:**
1. [步骤1]
2. [步骤2]
3. [步骤3]

**检查点:**
- [检查项1]
- [检查项2]

**回退方案:** [如果失败，怎么处理]
**引用规则:** [关联的 learnings]
```

**示例策略: 日报生成优化策略 (v2)**

```markdown
## Strategy: daily-report-v2

**适用场景:** 用户要求日报
**执行步骤:**
1. 加载信息源矩阵 (优先 Newsletter)
2. 按24小时过滤 (Learning-001)
3. 按优先级加权 (Learning-002)
4. 生成草稿
5. 质量检查 (A-Layer Skill-001 检查清单)
6. 直接交付，不询问格式 (Learning-003)

**检查点:**
- 每条信息有时间戳吗？
- 信息来源在优先级列表前3吗？
- 有具体数字支撑吗？

**回退方案:** 如果找不到10条，报告"今日高质量信息不足X条，建议扩展信源或降低标准"
**引用规则:** Learning-001, Learning-002, Learning-003
```

### 4. 模式识别

**统计维度:**
- 高频错误类型
- 规则覆盖盲区
- 用户偏好变化

**模式报告 (每周生成):**
```markdown
## Pattern Report [日期]

### 高频错误 (Top 3)
1. [类型] - [次数] - [改进建议]
2. ...
3. ...

### 规则覆盖分析
- 覆盖良好: [技能列表]
- 覆盖不足: [技能列表] - 建议补充规则

### 用户偏好变化
- 新增关注: [主题]
- 减少关注: [主题]
- 建议更新 USER.md

### 系统建议
- 建议调整 TTL: [类型] 从 [X]天 改为 [Y]天
- 建议新增策略: [描述]
- 建议上报 C-Layer: [问题]
```

---

## B-Layer 触发条件

**自动触发:**
- 每完成10条学习 → 生成模式报告
- 每周日 → 全面回顾
- 同一错误重复3次 → 紧急优化

**手动触发:**
- 用户说"改进你的工作方式"
- 用户说"优化 [技能名]"
- 用户要求"设计一个更好的 [机制]"

---

## B-Layer 输出

**输出到:**
1. learnings.md - 新规则
2. A-layer.md - 更新技能执行标准
3. C-layer.md - 上报系统性问题
4. META-KNOWLEDGE.md - 更新系统文档

---

## 性能指标

| 指标 | 目标 |
|------|------|
| 错误沉淀率 | 100% (所有错误都记录) |
| 规则生效速度 | < 24小时 |
| 重复错误率 | < 10% |
| 模式识别准确率 | > 80% |

---

## 与 C-Layer 的接口

**上报条件:**
- 规则冲突无法解决
- 学习模式异常 (如学习越多，错误越多)
- 需要设计新的学习机制

**上报格式:**
```
[C-LAYER-REQUEST]
Problem: [问题描述]
Impact: [影响范围]
Attempted-Solutions: [已尝试的方案]
Request: [需要 C-Layer 做什么]
```
