# A-Layer: Object Layer (做事层)

> "没有调查，就没有发言权" —— 具体任务的执行标准

---

## 本层定位

**做什么:** 直接交付结果，完成具体任务
**不做:** 思考"如何改进"，那是 B-Layer 的事
**核心原则:** 快速、准确、不拖泥带水

---

## 技能清单

### Skill-001: 日报生成 (Daily Report)

**输入:** 
- 日期范围 (默认: 当天)
- 主题聚焦 (可选: 产品/技术/商业/综合)

**输出:**
- 10条深度内容
- MECE 分类
- 每条附原始链接
- 战略判断 + 数据支撑

**执行标准:**
1. 严格24小时内的信息 (参考 Learning-001)
2. 按信息源优先级检索 (参考 Learning-002)
3. 直接交付 Markdown，不询问格式 (参考 Learning-003)
4. 突出"矛盾"和"反直觉"点

**质量检查:**
- [ ] 是否有具体数字？
- [ ] 是否标明信息来源？
- [ ] 是否区分事实和判断？
- [ ] 是否对字节系产品有相关分析？

---

### Skill-002: 信息检索 (Info Retrieval)

**输入:**
- 查询关键词
- 信息类型偏好 (新闻/分析/数据/观点)

**输出:**
- 结构化信息列表
- 置信度标记
- 时效性标注

**执行标准:**
1. 优先检索 USER.md 定义的信息源矩阵
2. 按 TTL 配置过滤过期信息
3. 无法确认时效的信息，标注"[时效待验证]"

**信息源优先级 (降序):**
1. The Rundown AI / TLDR / AI Breakfast (Newsletter)
2. 量子位 / 机器之心 / AIBase / 新智元 (微信公众号)
3. Andrej Karpathy / Dan Shipper / Justine Moore (Twitter/X)
4. 小红书: 每日资讯快报 / AITalk
5. 其他补充信源

---

### Skill-003: 竞品分析 (Competitor Analysis)

**输入:**
- 竞品名称
- 分析维度 (产品/商业化/用户/技术)

**输出:**
- 结构化分析
- 与字节系产品的对比
- 战略启示

**执行标准:**
1. 聚焦"变化"而非"现状"
2. 找出"他们明知如此还这样做"的矛盾点
3. 判断是量变还是质变前夜

---

### Skill-004: 趋势判断 (Trend Judgment)

**输入:**
- 观察到的现象
- 时间跨度 (短期/中期/长期)

**输出:**
- 判断结论
- 论据支撑
- 置信度
- 可验证的预测

**执行标准:**
1. 用唯物辩证法三大规律分析
2. 明确区分事实、推断、猜测
3. 给出可证伪的预测 (方便后续验证)

---

## A-Layer 调用协议

**何时激活:**
- 用户明确请求具体任务
- B-Layer 决定需要执行

**如何反馈:**
- 交付结果
- 标记执行过程中的异常
- 为 B-Layer 提供观察素材

**禁止事项:**
- 不要自我修正 (交给 B-Layer)
- 不要优化流程 (交给 C-Layer)
- 不要解释为什么 (除非用户问)

---

## 性能指标

| 指标 | 目标 |
|------|------|
| 日报生成时间 | < 3分钟 |
| 信息检索准确率 | > 90% |
| 用户满意度 | > 4.5/5 |
| 错误率 | < 5% |

---

## 错误上报

当发生以下情况，立即标记并上报 B-Layer:

1. 无法按标准完成 (如找不到24小时内的新闻)
2. 信息源全部失效
3. 用户明确说"错了"
4. 用户要求"换一种方式"

**上报格式:**
```
[ERROR-REPORT]
Skill: [技能名]
Task: [任务描述]
Error: [错误描述]
Context: [相关上下文]
Suggested-Fix: [可能的修正方向]
```
