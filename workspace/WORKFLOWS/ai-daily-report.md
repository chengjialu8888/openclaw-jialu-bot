# AI日报自动化工作流配置
# 目标：每天自动收集、筛选、验证AI行业动态，生成高质量日报

## 核心原则
1. **严格时效性**：只收录24小时内发布的真实新闻
2. **源头验证**：交叉验证多个信源，排除旧闻和重复内容
3. **质量优先**：宁缺毋滥，无足够新鲜内容时输出"今日无重大动态"

---

## 时间验证规则（关键）

### 发布时间判定标准
- **可信来源**：官方博客、公司新闻稿、权威科技媒体（TechCrunch、The Verge、机器之心、量子位等）
- **验证方法**：
  1. 优先查看官方发布渠道（公司博客、官方Twitter/X账号）
  2. 交叉对比多个独立信源的报道时间
  3. 检查文章中的具体日期表述（如"今天"、"昨日"、具体年月日）
- **排除项**：
  - 网页索引/更新时间 ≠ 新闻发生时间
  - "回顾"、"盘点"、"年度总结"类文章
  - 未标注明确发布时间的文章

### 搜索策略
```
搜索关键词模板：
- "AI news [today's date]"
- "[Company] announced [today's date]"
- "[Product] launched March 2026"

时间筛选参数：
- --time-range 1d（必须）
- 优先查看官方来源
- 交叉验证至少2个独立信源
```

## 信息源分级

### S级（官方信源，最优先）
- 公司官方博客/新闻稿
- 官方Twitter/X账号
- 官方GitHub release页面
- 官方YouTube频道发布

### A级（权威科技媒体）
- TechCrunch、The Verge、Ars Technica
- 机器之心、量子位、雷锋网
- 36氪、虎嗅（科技板块）

### B级（行业垂直媒体）
- VentureBeat、AI News
- InfoQ、CSDN

### 排除级（需谨慎）
- 内容聚合网站（今日头条、百家号等无原创内容的平台）
- 未标注明确来源的转载文章
- 论坛帖子、自媒体猜测性内容

---

## 日报内容标准

### 收录标准
1. **产品发布**：新模型、新功能、新产品上线
2. **重大更新**：现有产品的重大版本更新（需有明确版本号或功能清单）
3. **商业动态**：融资、并购、战略合作、重大合同
4. **政策监管**：政府政策、行业监管变化
5. **数据发布**：用户增长、收入、市场份额等关键数据

### 排除标准
1. **旧闻新炒**：超过24小时的新闻
2. **传闻/猜测**：未经官方确认的消息
3. **常规运营**：日常维护、小bug修复
4. **过度解读**：分析师观点、市场预测（除非有重大突破）

---

## 验证清单（每篇报道必须检查）

- [ ] 是否有明确的官方来源链接？
- [ ] 发布时间是否在24小时内？
- [ ] 是否为原创内容而非转载？
- [ ] 是否包含具体的事实/数据而非泛泛而谈？
- [ ] 是否与其他信源交叉验证？

---

## 输出格式

### 时效性标注
```
✅ 已验证 - 24小时内发布，有官方来源
⚠️  待验证 - 信源单一，需进一步核实
❌ 旧闻 - 超过24小时或无法验证时效性
```

### 内容结构
1. **今日头条**（1-3条，必须是最重大且已验证的新闻）
2. **产品动态**（已验证的新功能/新产品）
3. **商业动态**（融资/并购/合作）
4. **国内动态**（国产AI相关，需额外核实）
5. **今日洞察**（基于当日新闻的分析）

---

## 错误处理机制

### 当缺乏足够新鲜内容时
- **诚实输出**："今日（3月X日）AI行业重大动态有限，主要新闻集中在X月X日"
- **不编造**：宁可少报，不多报
- **说明原因**：解释为什么内容较少（如周末、节假日、美国时间尚早等）

### 当信息冲突时
- **优先官方**：以公司官方声明为准
- **标注争议**：如果存在不同说法，标注"信息存在争议，以官方为准"

## 定时执行配置

```yaml
schedule:
  # 每天上午10:00执行（北京时间）
  # 此时美国时间为前一天晚上，大部分新闻已发布
  - cron: "0 10 * * *"
  
  # 可选：下午6:00补充更新（北京时间）
  # 覆盖美国当天工作时间的发布
  - cron: "0 18 * * *"

timezone: Asia/Shanghai

actions:
  - name: search_and_verify
    tool: coze-web-search
    params:
      time_range: "1d"
      sources_priority: ["official", "tier1_media", "tier2_media"]
      cross_verify: true
      
  - name: generate_report
    template: "daily_ai_report.md"
    
  - name: review_and_send
    require_approval: false  # 设置为true如果需要人工审核
```

## 自检提示词（每次生成日报前必须读取）

```
在开始收集今日AI新闻前，请确认：

1. 当前日期是？（YYYY-MM-DD格式）
2. 美国当前日期是？（考虑时差）
3. 搜索时使用 --time-range 1d 参数了吗？
4. 每篇新闻是否检查了原始发布时间（非网页索引时间）？
5. 是否排除了"回顾"、"盘点"、"年度总结"类文章？
6. 是否交叉验证了至少2个独立信源？
7. 如果今日新闻不足，是否准备诚实说明而非填充旧闻？

确认以上问题后再开始生成日报。
```
