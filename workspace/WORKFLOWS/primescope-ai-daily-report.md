# Prime Scope AI 日报工作流

> 融合 Prime Scope 的结构化分类与手写版的标签化简洁
> 目标：3分钟速读 + 数据支撑 + 结构性洞察

---

## 一、内容维度（3大板块）

| 维度 | 聚焦点 | 收录标准 |
|------|--------|----------|
| **大厂动向** | OpenAI、Google、Anthropic、Meta、字节、阿里等 | 产品发布、模型更新、重大商业动态 |
| **初创动向** | AI native 创业公司 | 融资（500万+）、产品发布、模式创新 |
| **生态动向** | 监管、基础设施、开源、安全 | 政策变化、标准更新、重大安全事件 |

**无内容处理：**
- 如某维度无动态，标注"今日无重大动态"
- 避免读者误以为遗漏

## 二、单条新闻格式模板

### 基础模板
```
【厂商 - 产品/事件（核心价值标签）】[链接标题](URL) 一句话摘要（含关键数据）。可信度标识
```

### 核心价值标签（供选择）
| 标签 | 适用场景 |
|------|---------|
| **优化C端体验** | 消费级产品更新、交互改进 |
| **优化B端成本** | 降价、提速、企业功能 |
| **技术突破** | 新模型、新架构、性能提升 |
| **商业化进展** | 收入、融资、用户增长 |
| **安全风险** | 漏洞、隐私、合规问题 |
| **人事变动** | 核心人员离职/加入 |
| **生态扩张** | 合作、开源、平台化 |

### 可信度标识
| 标识 | 含义 | 示例来源 |
|------|------|---------|
| ✅官方 | 官方博客/推特/新闻稿 | OpenAI Blog、Google AI |
| ✅权威 | 权威科技媒体 | TechCrunch、彭博、The Verge |
| ⚠️垂直 | 垂直AI媒体 | AIbase、机器之心、量子位 |
| ⚠️传闻 | 社交平台/匿名消息 | X平台、知情人士 |

### 数据呈现规范
**必须量化：**
- 性能："幻觉率降低26.8%"
- 定价："$0.25/100万输入tokens"
- 速度："TTFA快2.5倍"
- 规模："DAU突破1亿"
- 财务："年收入run-rate超25亿美元"

**避免模糊形容词：**
- ❌ "大幅提升" → ✅ "提升45%"
- ❌ "显著降低" → ✅ "降低26.8%"
- ❌ "快速增长" → ✅ "收入增长158.9%"

```markdown
## 💡 今日洞察（Key Takeaways）

**生成原则：**
- 不是新闻汇总，是**跨事件的结构性判断**
- 找**矛盾和张力**（对立统一规律）
- 每个洞察必须包含：**现象 → 判断 → 对字节启示**

**洞察类型（3选2-3）：**

### 类型1：关键张力（对立统一）
找到今日新闻中看似矛盾但共存的现象，分析其内在联系。

**格式：**
```
【关键张力X】"A"与"B"的拉锯
- 现象：列举体现张力的具体新闻（2-3条）
- 判断：用唯物辩证法分析（张力是变化的能量，离质变还差多少？）
- 对字节启示：具体到产品/战略层面（不要泛泛而谈）
```

**示例：**
```
【关键张力1】"安全"与"商业"的拉锯
- 现象：OpenAI Pentagon合同被迫回溯（监控争议）；Anthropic拒绝Pentagon后收入翻倍至200亿美元
- 判断：to B军单收入好赚但品牌损耗高，"安全立场"本身可以成为商业差异化
- 对字节启示：谨慎评估政府/军方合作，将"数据安全"作为出海核心卖点
```

### 类型2：趋势判断（质量互变）
从多个事件中找到正在形成的新趋势，判断其发展阶段。

**格式：**
```
【趋势判断X】XXX成为新战场/新常态
- 信号：列举支撑趋势的多个信号（至少2个独立来源）
- 判断：当前处于什么阶段（早期/爆发前夜/已主流化）
- 对字节启示：字节在该趋势中的位置、机会窗口、行动建议
```

**示例：**
```
【趋势判断1】"语音交互"成为开发者工具新战场
- 信号：OpenAI、Anthropic同日发布Codex/Claude Code语音功能；Google Pixel集成语音代理
- 判断：处于早期普及阶段，90后/00后开发者接受度更高，离主流化还有12-18个月
- 对字节启示：Trae可集成语音编程能力；豆包可探索语音代码审查场景；窗口期6-9个月
```

### 类型3：战略启示（否定之否定）
从竞争对手的动作中找到字节应该避免或学习的点。

**格式：**
```
【战略启示X】竞争对手的XXX策略值得警惕/借鉴
- 对手动作：具体描述竞争对手的举措
- 深层意图：分析其背后的战略考量（不要只看表面）
- 对字节启示：字节是否需要跟进、如何差异化、风险是什么
```

**示例：**
```
【战略启示1】Google的"端侧下沉"策略值得警惕
- 对手动作：Gemini能力下沉到Pixel（代理任务）和Google Home（实时搜索）
- 深层意图：绕过云端模型同质化竞争，建立"端+云"协同壁垒
- 对字节启示：加速豆包在手机、智能家居场景的端侧部署；避免纯云端竞争
```

**洞察数量：**
- 新闻丰富时：2-3个洞察（张力、趋势、启示各1个）
- 新闻较少时：1-2个深度洞察（宁缺毋滥）

**质量检查：**
- [ ] 洞察是否跨越多个新闻事件？（单一事件→不算洞察）
- [ ] 是否包含"对字节启示"？（没有→不完整）
- [ ] 判断是否可证伪？（不可证伪→太模糊）
- [ ] 是否用了经典智慧框架？（易经察势/孙子兵法/资本论等）

---

📊 **日报统计**
- 信息源：8条（✅官方3条 / ✅权威2条 / ⚠️垂直3条）
- 阅读时间：约3分钟
- 大厂覆盖：OpenAI、Google、Anthropic
- 关键数据点：7个
```

## 五、生成前检查清单

### 基础检查
- [ ] 当前北京时间：____年__月__日
- [ ] 使用了 `--time-range 1d` 参数
- [ ] 每篇新闻是否标注了核心价值标签？
- [ ] 关键数据是否量化（避免"大幅提升"等模糊词）？
- [ ] 来源是否标注可信度（✅/⚠️）？

### 洞察检查（关键）
- [ ] 今日洞察是否跨越多个新闻事件？（单一事件→退回重写）
- [ ] 是否识别了矛盾/张力（对立统一）？
- [ ] 是否判断了趋势阶段（质量互变）？
- [ ] 是否包含"对字节启示"？（没有→不完整）
- [ ] 判断是否可证伪？（"AI改变世界"→太模糊；"语音交互12-18个月主流化"→可证伪）
- [ ] 是否用了经典智慧框架？（易经察势/孙子兵法/资本论等）

### 字节视角检查
- [ ] 是否涉及字节直接竞争对手的动态？
- [ ] 是否分析了竞争格局变化对字节的影响？
- [ ] 是否给出了actionable的建议（而非泛泛而谈）？

## 六、质量红线

1. **宁缺毋滥** — 无足够新鲜内容时诚实说明
2. **严格24小时** — 超过24小时的新闻不收录
3. **官方优先** — 公司博客 > 权威媒体 > 垂直媒体
4. **数据必核** — 无法验证的数据标注"待核实"
5. **信源交叉** — 传闻类信息至少2个独立信源
